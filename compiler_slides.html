Our Simple Language

def fib n if n < 2 return n end a = fib n - 1 b = fib n - 2 return a + b end main = fib 9 print main

Features: functions, conditionals, arithmetic, recursion

Step 1: Parsing

Convert source code into an Abstract Syntax Tree (AST)

Source:

def fib n return n end

AST:

{ "type": "def", "name": "fib", "arg": "n", "body": [ { "type": "return", "expr": "n" } ] }

Parser Implementation

def parse(lines): ast = [] stack = [ast] # Stack for nested structures for line in lines: if re.match(r"^def (\w+) (\w+)$", line): # Create function node func = {"type": "def", "name": name, "arg": arg, "body": []} stack[-1].append(func) stack.append(func["body"]) # Enter function body elif line == "end": stack.pop() # Exit current scope # ... handle other constructs

Uses a stack to track nested scopes

Step 2: Compilation

Transform AST into bytecode instructions

Expression: "fib n - 1"

Becomes:

SUB n 1 tmp2 # tmp2 = n - 1 PUSH tmp2 # Push argument CALL fib # Call function MOV _retval tmp3 # Store result

Function Call Recognition

# Global set to track all defined functions function_names = set() # During parsing, collect function names if re.match(r"^def (\w+) (\w+)$", line): function_names.add(function_name) # During compilation, recognize calls words = expr.split() if words[0] in function_names: # This is a function call! compile_function_call(words[0], words[1:])

Key insight: Collect all function names first, then recognize calls

Step 3: Virtual Machine

A stack-based VM executes our bytecode

Key Components:

  • Program Counter (PC)
  • Call Stack
  • Environment Stack
  • Label Map

Instructions:

  • MOV, ADD, SUB
  • CALL, RET
  • JMP, JGE
  • PUSH, PRINT

Function Call Mechanism

PUSH arg # 1. Push argument CALL function # 2. Jump to function PARAM arg # 3. Bind parameter ... function body ... RET value # 4. Return with value # 5. Resume here with result in _retval

Uses environment stack for local variables

Each function call gets its own scope

Mutual Recursion Example

def is_even n if n < 1 return 1 end return is_odd n - 1 end def is_odd n if n < 1 return 0 end return is_even n - 1 end

is_even calls is_odd, and vice versa!

Works because we collect all function names during parsing

Execution Trace: is_even(2)

[PC=15] PUSH 2 → push argument => 2 [PC=16] CALL is_even → pop arg => 2, jump to is_even [PC=01] PARAM n → PARAM n = 2 [PC=02] JGE n 1 tmp1_end → JGE 2 >= 1, jumping to tmp1_end [PC=05] PUSH 1 → push argument => 1 [PC=06] CALL is_odd → pop arg => 1, jump to is_odd [PC=09] PARAM n → PARAM n = 1 [PC=10] JGE n 1 tmp2_end → JGE 1 >= 1, jumping to tmp2_end [PC=13] PUSH 0 → push argument => 0 [PC=14] CALL is_even → pop arg => 0, jump to is_even

Deep recursion with multiple function calls!

Key Insights

  • Two-pass approach: Parse first, then compile
  • Symbol collection: Gather function names early
  • Stack-based execution: Simple but powerful
  • Environment stacking: Handles scope naturally
  • Mutual recursion: Falls out naturally from design

This architecture scales to much more complex languages!
